# zooba 商談パイプラインレポート

GitHub Pages: https://sawasaka.github.io/zooba-pipeline-report/
リポジトリ: https://github.com/Sawasaka/zooba-pipeline-report

## プロジェクト概要

zooba の商談パイプラインを可視化するCEO向けエグゼクティブレポート。
単一HTMLファイル（`index.html`）で完結。`executive_summary.html` は同一内容のコピー。

## デザイン仕様

- トンマナ: ネイビー(#0D1B2A) × ゴールド(#C9A84C)のエグゼクティブスタイル
- 見出し: Hiragino Mincho ProN（明朝体）
- 本文: Hiragino Sans（ゴシック体）
- KPI数値のカウントアップアニメーション、フェードインアニメーション付き

## ステージ定義

| ステージ | CSSクラス | 色 | KPIラベル |
|---|---|---|---|
| PJ化予定あり | `status-ph3` | gold (#C9A84C) | PJ化予定あり |
| PJ可能案件 | `status-ph4` | blue (#5B8DB8) | PJ可能案件 |
| PJ進行中 | `status-ph5` | green (#2ECC71) | PJ進行中 |

## ビジネスルール

### 月別帰属

商談は**初回登録月**に帰属する。3月に登録した商談は、4月以降にステージ変更があっても3月タブ内で更新する。新規企業は現在月のタブに追加する。

### KPIウォーターフォール（累積カウント）

KPIカードの数値は「その段階を**通過した累積数**」。上位ステージに移行しても下位の数値は減らさない。

```
有効商談数 ≥ PJ化予定あり ≥ PJ可能案件 ≥ PJ進行中
```

ステージ移行例：1社が「PJ化予定あり」→「PJ進行中」に移行した場合、途中の「PJ可能案件」も経由するため：
- PJ化予定あり: 変化なし（すでにカウント済み）
- PJ可能案件: +1
- PJ進行中: +1

### 移行率の計算

- 最終転換率 = PJ進行中 ÷ 有効商談数
- 各段階の移行率 = 次段階の累積数 ÷ 当該段階の累積数
- 小数点以下切り捨て、%表示

### ツールチップの企業リスト

各KPIカードのホバーツールチップには、**現在そのステージにいる企業のみ**を表示する（上位に移行済みは除く）。該当なしの場合は「該当なし」と表示。

## 更新後の手順

1. `index.html` を更新
2. `executive_summary.html` にコピー
3. git commit & push（GitHub Pagesに自動デプロイ）
